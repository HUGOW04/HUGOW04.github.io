Key Features:

  • File Encryption: Encrypts files using a custom algorithm to prevent unauthorized access.
  • File Decryption: Provides a method to decrypt files with the corresponding decryption key.
  • Exclusions: Avoids encrypting critical system directories to prevent system damage.
  • Progress Reporting: Outputs status updates on file encryption and decryption.
  • Windows API Integration: Utilizes Win32 API for file operations and system interactions.

How It Works:

The ransomware encrypts files in user directories by reading and modifying them in blocks. Encrypted files are appended with a `.tu` extension. The decryption process reverses the encryption, restoring files to their original state using the decryption key. Critical system files and directories are excluded from encryption.

Installation and Usage:

Compile the C++ source code in a Windows environment and run the executable. Use the ransomware in a controlled, educational setting. Encrypted files will have a `.tu` extension; use the decryption tool to restore them. Ensure the ransomware is used responsibly to avoid unintended consequences.
